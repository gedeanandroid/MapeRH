# Workflow: Atualizar Documentação

Sempre que implementar alterações significativas no código, a documentação em `docs/DOCUMENTATION.md` deve ser atualizada.

## Quando Atualizar

1. **Novos componentes ou páginas** - Adicionar na seção "Estrutura de Arquivos" e "Módulos Funcionais"
2. **Novas tabelas ou colunas no banco** - Atualizar o diagrama ER e descrição de tabelas
3. **Novas rotas** - Adicionar no mapa de rotas
4. **Novos serviços ou contexts** - Documentar na seção "API e Serviços"
5. **Novas Edge Functions** - Adicionar na seção "Edge Functions"
6. **Alterações em roles/permissões** - Atualizar seção de autenticação

## Como Atualizar

1. Abra o arquivo `docs/DOCUMENTATION.md`
2. Localize a seção relevante para a alteração
3. Faça as atualizações necessárias mantendo o padrão existente
4. Atualize a data no rodapé do documento

## Checklist de Atualização

- [ ] Estrutura de arquivos atualizada
- [ ] Modelo de dados atualizado (se aplicável)
- [ ] Rotas atualizadas (se aplicável)
- [ ] Serviços documentados (se aplicável)
- [ ] Data de atualização no rodapé
